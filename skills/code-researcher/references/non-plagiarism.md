# Non-Plagiarism Rules

This skill must be original in wording, structure, examples, and workflow.

Allowed:

- General engineering ideas such as evidence-backed analysis, architecture
  diagrams, source tracing, tradeoff discussion, and maintainer-oriented notes.
- Independent conclusions derived from the target repository's source code.
- Widely known terminology such as entry point, dependency, module, API, runtime
  path, abstraction, and architecture.

Not allowed:

- Copying prose from another skill or report.
- Reusing another skill's section sequence, table layout, prompt templates, or
  examples with light paraphrasing.
- Translating another skill's text and presenting it as original.
- Preserving distinctive phrases, metaphors, or checklist wording from another
  skill.

When inspired by another tool, convert the inspiration into an independent
requirement. For example, "important claims need code evidence" is a general
principle; the exact checklist and phrasing from another implementation should
not be reused.

The identity of this skill is the four-layer method:

1. Codebase Map
2. Runtime Story
3. Design Inquiry
4. Maintainer Notes

Keep new additions consistent with that identity.
